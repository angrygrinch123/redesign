Are we Actually Accessible?

We pride in our advocacy for inclusion, diversity, and accessibility, but do we actually demonstrate that? The website says otherwise.

Despite being the face of Brown's CS program, the Brown CS PhD website has an unintiutive, inaccessible design that makes it hard for prospective applicants and current PhD students find information relevant to the program. The main focus of the project is to change the visual hierarchy and add more alt text for visual accessibility. So the following is a more enhanced narrative of the first iteration of this project with my personal reflections on the project and summary of feedback.

The first iteration suffered from lack of clear narrative, not enough visual contrast, and images that were too large. Link to First Iteration of the Project.

Key Issues

The main issues of the current website came from the lack of visual hierachy, lack of alt text, and poor visual contrast. It was not clear who the website was made for because all the content is concatenated on one page. There needs to be a better segregation of the page based on user type - Current PhD Students vs Prospective Students would want different kinds of information.

  • Layout Alignment - The alignment does not follow Gesalt Principles as there is no continuity in the style of the website nor is the content grouped by similarity; so the layout does not follow the conceptual model that users usually expect (i.e. prospective students and current students expect separate pages or dividers to find relevant info and not to have all the info one one page)
  • Poor Contrast Management - Too much bolding, bullet pointing, and navigation bars create confusion about what is truly important on the page
  • Inefficient - Does not follow the conceptual model of prospective students nor current students; most expect a separate section for different users and the order in which they find this information usually is more organized; this also affects memorability because the user must re-adjust each time and relearn the format and information hierarchy
  • Visual Overload - There are too many options to navigate around the page (i.e. the navigation bar, the bread crumbs, and the tabs) which creates a high bar for learnability
  • Inaccessible - Much of the alt texts are missing and the color contrast is not friendly to users with visual impairments

WAVE Findings

The biggest change I wanted to make was (1) Change of Hierarchy, (2) Addition of Alt Text, (3) and Visual Contrast. This was to create a easier navigation experience based on user profile (i.e. Current vs Prospective PhD Student) as well as highlight important dates and contact information.

  • 16 Contrast Errors - The orange to white background signal a lot of contrast error
  • 2 Suspicious Alternative Textst - The alt text don't have any meaningful descriptions in it and only includes the picture name

Lo-fi Redesign

Low fidelity redesigns for website, smartphone, and tablet. The new wireframes reduce visual overload by removing multiple navbars, centralizes the layout alignment into one central flow, adds better support for users with visual impairments with added contrast and alt texts, and increases efficiency by segregating content with a hierarchical model by users.

Visual Design Style Guide

For the design style guide, I use bootstrap for the aesthetic choice. They typography is the embedded ones form bootstrap. Many of the components are from: Figma Template for Boostrap and Bootstrap Components I chose very simple, but bold colors to add contrast between texts and link. The main point of the navbar was to enhance visual hierarchy and the simple, contrasting colors to be accessible to those with visual impairments.

Hi-fi Redesign

High fidelity redesigns for website, smartphone, and tablet. The new wireframes demonstrate dynamic realignment of layout and responsive changes of colors over hover functionalities.

Redesign Iterations

In addition, to the change in narrative, I incorporated the feedback from a peer evaluation session. (1) Add Margins (2) Central View Flow (3) Accordions to replace the Boxes for Responsive-ness (4) Fewere color schemes for heightened visual contrast.
